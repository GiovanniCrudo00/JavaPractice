ABSTRACT FACTORY(AF)
Provvede un' interfaccia per creare famiglie di oggetti correlati o dipendenti
sensa specificare le loro classi concrete

Gestisce look-and-feel diversi per i vari sistemi operativi, per essere portabile un' applicazione
non dovrebbe avere codice eccessivo per particolari look-and-feel

La classe widget factory prende il ruolo di creatore con metodi virtuali che a seconda del sistema operativo
richiamano i costruttori di scrollbar, window etc.. che siano tutti della stessa famiglia.

Conseguenze
Le classi concrete non si mescolano mai perchè vengono selezionate a priori e il pattern tiene distinte le
linee di gerarchie delle classi.
Il cliente manipola le classi tramite le interfacce astratte.
I nomi delle classi non compaiono nel codice cliente.

Questo pattern rende facilmente modificabili le famiglie.
Tutti i ptodotti saranno dello stesso tipo e coerenti tra loro.

Il pattern rende difficile aggiungere nuovi prodotti perchè si dovrebbero modificare tutte le sottoclassi della
fabrica stessa.

Gli oggetti possono essere creati tramite metodi parametrici ma questo crea problemi di class casting quando si vogliono usare
metodi specifici e non della superclasse.

SINGLETON(SI)

Assicura che una classe abbia solo un istanza, con punto di creazione centralizzato con accesso globale.
Una variabile globale rende l' oggetto accessibile ma non ti concede di istanziarlo più volte la classe è responsabile
della sua unica istanza.

Si utilizza se il punto di accesso all' istanza è ben definito e conosciuto, anche se la sola classe può essere usata 
per istanziare sottoclassi.

In java essendo protected solo a livello di package si deve creare un insieme di sottopackage per gestire il costruttore
protected effettivo.
